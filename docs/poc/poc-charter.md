# 七日 PoC 游戏合同

状态：玩法范围已确认，待实现验证
性质：非正史、内容与数值可丢弃、只验证规则；承载它的 Harness 按生产级维护
目标体验时长：单轮 25–45 分钟

## 1. PoC 要回答的问题

这个 PoC 不证明“完整游戏能做完”，只回答五个高风险问题：

1. 一周是否能形成可读的经营压力与恢复空间；
2. 行动点和体力是否产生不同取舍，还是重复限制；
3. 亲自、部分委托、完全委托、计划停业是否各有用途；
4. 经营、关系和调查是否争夺同一时间，而不是三条并列清单；
5. 玩家能否从预测与结算中理解结果为何发生。

## 2. 固定边界

- 一个周循环：周一到周日；
- 周一至周六营业，周日默认店休并结算城镇重建税负；
- 上午、下午、晚上三个时段；
- 每时段有行动点，行动另消耗体力；
- 两种生活方针；
- 四道菜、五种原料、两个客群；
- 一名无个人剧情的伙计；
- 四种营业方式；
- 一个设施二选一；
- 一个关系机会，包含可选的`生气`与道歉路径；
- 一个战争线索文字冒险和一次固定种子的 2D6 检定；
- 一个确定性属性门槛；
- 一次周末税负；
- 一个滚动自动存档、一个 Quick Save 和一个手动存档槽；
- 决策前预测、结算后拆账、开发者面板和结构化日志。

新局 Snapshot 已位于周一上午；`run.start` 先物化 D1 demand 并展示开局文字卡。玩家完成该 Scene 后、执行任何
普通 D1 行动或推进时段前必须选择生活方针；未选择时不能继续，不存在 day 0 或隐藏默认值。

## 3. 有意制造的主要取舍

周五上午同时展示关系机会与文字冒险，并明确它们共享一个机会窗口：

- 文字冒险必须上午出发，并在下午完成；
- 未出发时，关系事件在下午正式开放；
- 一起修理消耗整个下午，主动放弃或冲突选项不消耗 AP，但都会提交本周关系结果；
- 选择其中一个会让另一个本周失效；
- 晚间仍可经营，但冒险后的体力消耗可能迫使玩家委托、休息或降低投入。

因此一轮可以维持酒馆并完成关系或调查之一，不能把经营、关系和调查全部收满。这是 PoC 对“取舍来自选择”的直接验证。正式游戏后期可以通过设施、能力或周目知识挑战更高完成度。

## 4. 不进入 PoC

- 正式序章与老人的完整演出；
- 第二周和长期日历；
- 属性成长、技能树、完整关系阶段转移；
- 多地点冒险、战斗、装备；
- 天气、季节和价格波动；
- 完整设施树与员工等级；
- 多周目、跳过、Mod、正式美术、3D；
- 运行时 LLM、网络服务、账号和云存档；
- 成人内容。

## 5. 成功、危险与失败

### 5.1 经营结果

- **稳定完成**：缴清税负，税后现金至少 20，人气至少 50，并建成一个设施；
- **危险完成**：能够缴税，但不满足全部稳定条件；若人气低于 45，额外标记`声誉危机`原因，但仍属于可恢复的危险完成；
- **欠税失败**：周日可用现金不足以缴税。

从周五开始显示税后保守预测。失败必须能由账本解释；不允许隐藏 flag 直接判负。

### 5.2 关系与故事结果

它们是独立记录，不改变经营成功定义：

- 关系：未参与（维持经营伙伴）、完成、主动放弃、冲突后和解、冲突未处理；
- 调查：未进行、受挫、带代价成功、完整成功、卓越成功。

PoC 不包含女主离开或关系 Bad End；轻微冲突最多产生临时`生气`和小额好感变化。

## 6. 教学开放顺序

- 周一：精确需求；只教采购、备菜、菜单、亲自营业和拆账；
- 周二：开放部分委托、确定性属性门槛和 `±1` 需求预测；
- 周三：开放完全委托、计划停业和税负义务追踪，只显示税额、现有现金与当前缺口，不预测未来收入；
- 周四：设施二选一；
- 周五：关系/调查互斥机会、2D6 与基于已提交当日计划的保守税后预测；
- 周六：用选择后果回流经营，提供最后恢复窗口；
- 周日：无营业，缴税、总结、导出诊断并可立即重开。

## 7. 玩家信息合同

每项可执行命令在确认前显示：

- 确定的现金、行动点和体力成本；
- 预计收益或范围；
- 会占用的时段；
- 会失效的互斥机会；
- 重大缺货、欠税或状态风险。

每次营业结算显示：

- 两类客群及其有效订单；
- 计划份数、接待/备菜容量与实际销量；
- 缺货和未售成品；
- 收入、采购历史、工资、营业杂费、腐败和税负；
- 人气、体力、默契、好感和 Aura 变化及原因。

## 8. 通过闸门

在开始 Unity 或正式内容生产前，PoC 至少满足：

- 固定种子的完整一周自动化测试可重复；
- 多种子策略的精确通过率满足 `balance-v0.md` 第 14 节，不使用“大多”“通常”作为机器验收；
- 读档后需求、检定、库存和结算完全一致；
- AP、体力、现金以外的库存数量不出现非法负数；
- 至少 5 名未参与设计的试玩者中，4 人能说明本周选择了什么、主动放弃了什么；
- 普通日的必要决策类别不超过 3 个，D5 不超过 4 个；
- 玩家能从拆账页指出亏损或增长的主要原因。

若同一策略在绝大多数种子中成为严格最优、完全委托成为陷阱、或玩家通过依次点击所有动作即可完成全部内容，先调整核心循环，不增加新系统。

“决策类别”按采购数量、备菜/休息分配、菜单与营业方式、剧情机会分别计数；同一类别内修改数量、复制昨日设置、查看预测和最终确认不重复计数。教程提示和纯信息弹窗不计数。
